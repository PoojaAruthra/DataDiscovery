Portfolio Products

are marketable offerings provided by teams to their stakeholders.
Portfolio Products can include multiple Insights. Insights have a clear value definition and stakeholder focus.

Insights

You can check out Insights by discovering them based on their Portfolio Product and by searching name, description, or tags.
Tags

Tags provide details to Insights and link them with Portfolio Product. To make sure a user can find the correct Insights of each Portfolio Product, create or add a tag with the name of the product and asign it to the insight. If the product changes its name, simply update the name of the product and the tag.

Every Tag has an associated type to add meaning. The following types are defined:
  • Product: Gives the Product Cluster of the Insight
  • Department: Target Department or Stakeholder
  • CreatorTeam: Team that was involved in the creation
  • ServiceType: Type of insight that is being offered, Dashboard, Notification, Consulting, Self-service,...
  • Technology: Technology used for hosting the Insight (e.g. Tableau, PowerBI)
  • Value: Does the Insight improve Efficiency (How), Effectivity (What), or Opportunity?
  • Phase: The phase of software engineering the insight is aimed at: [Define, Design, Implement, Test, Deploy, Operate]
  • Role: The role key stakeholder addressed in the insight
